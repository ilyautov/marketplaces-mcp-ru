# Как внести вклад

Спасибо за интерес к `marketplaces-mcp-ru`! Проект открытый (MIT), и репорты
багов, правки каталогов и сценарии приветствуются.

## С чего начать

```bash
git clone https://github.com/ilyautov/marketplaces-mcp-ru.git
cd marketplaces-mcp-ru

# офлайн-тесты (ключи не нужны) — все должны быть зелёные
env -u OZON_CLIENT_ID -u OZON_API_KEY -u WB_API_TOKEN python3 -m pytest tests/ -q

# selfcheck серверов
python3 serve.py wb --selfcheck
python3 serve.py ozon --selfcheck
python3 serve.py ozon-perf --selfcheck
```

Нужен Python 3.10+. Зависимости (`mcp`, `httpx`, `pyyaml`) `serve.py` ставит сам
в локальный `.venv` при первом запуске.

## Что особенно полезно

- **Боевая выверка методов.** Каталоги импортированы из OpenAPI-спеков: пути
  надёжны, но **HTTP-глаголы у импортированных методов не выверены**. Если live
  показал, что метод на самом деле `POST`, а не `GET` (или наоборот) — поправьте
  `*/endpoints.yaml` и приложите доказательство (код ответа, ссылку на доку).
- **Новые сценарии (workflows)** в `*/workflows.yaml` — пошаговые рецепты с
  трактовкой и типичными ошибками. Каждый шаг должен сверяться с каталогом.
- **Расширение safety-классификации** (`read` / `write` / `destructive`), если
  метод классифицирован слишком мягко или слишком строго.

## Правила

- **Никогда не коммитьте секреты.** Боевые ключи, `cabinets.json`, `.env` — мимо
  репозитория (`.gitignore` это закрывает, но проверяйте `git diff` перед
  коммитом). Утечку — по процедуре `SECURITY.md`.
- **Не ломайте офлайн-тесты.** Перед PR прогоните `pytest tests/ -q` и
  `--selfcheck` всех серверов.
- **Изменили число методов/тулов** — обновите цифры в `README.md`.
- Коммиты и PR — по-русски или по-английски, в повелительном наклонении
  («Add …», «Fix …», «Поправь …»).
