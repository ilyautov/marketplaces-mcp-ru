# Безопасность

`marketplaces-mcp-ru` работает с **боевыми API-ключами** ваших кабинетов
Wildberries и Ozon. Эти ключи двигают цены, остатки и деньги — относитесь к ним
как к паролю от расчётного счёта.

## Как устроено хранение ключей

- Ключи хранятся **только локально**, в `~/.marketplace-mcp/cabinets.json` с
  правами `chmod 600` (читает лишь ваш пользователь).
- Ключи **не пишутся** в конфиг MCP-клиента (`claude_desktop_config.json` и
  аналоги): туда идут только команды запуска сервера, без секретов.
- Ключи **не попадают в чат**: тул `*_check_auth` сообщает лишь факт наличия
  кредов, не печатая их.
- В репозиторий секреты не коммитятся: `.gitignore` закрывает `cabinets.json`,
  `.env` и `*.env`. История git проверена — токенов в ней нет.

## Что вы должны делать сами

- **Не вставляйте ключи в чат с агентом.** Вводите их только через установщик
  (`install.py` / `install.bat` / `install.command`) или файл `.env` локально.
- **Используйте API-ключи с минимально нужными правами.** На WB токен можно
  ограничить категориями; на Ozon заведите отдельный ключ под этот инструмент.
- **Ротация.** Если ключ мог засветиться (скриншот, лог, общий экран) —
  перевыпустите: WB → seller.wildberries.ru → Настройки → Доступ к API;
  Ozon → seller.ozon.ru → Настройки → API-ключи.
- Помните: это **alpha** и инструмент для агента. Любое решение, которое трогает
  цены, остатки или деньги, остаётся за вами — safety-гейт лишь не даёт сделать
  это случайно.

## Сообщить об уязвимости

Нашли проблему с безопасностью (утечка ключей, обход safety-гейта, инъекция в
вызов API) — **не открывайте публичный issue**. Напишите на
**ilyautov@gmail.com** с темой `SECURITY: marketplaces-mcp-ru`. По возможности
приложите шаги воспроизведения и версию. Постараюсь ответить в течение
нескольких дней.
